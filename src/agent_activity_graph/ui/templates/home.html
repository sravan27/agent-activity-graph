Reference Product

Enterprises need a runtime truth layer for agents inside business workflows.

Agent Activity Graph is a compact reference implementation for the missing evidence layer: what the agent did, when it did it, under whose authority, in which process step, against which systems, and whether policy or humans altered the path.

It is intentionally narrow. One invoice approval workflow. One local database. One policy engine. One replay surface. The point is not product breadth. The point is proving the layer.

What This Proves
AI workflow systems are incomplete when they stop at orchestration and do not preserve runtime evidence as a first-class object.
Policy cannot be a hidden branch in agent code. It needs an auditable decision record with violated rules, explanations, and recommended next action.
Replay matters because debugging an agentic process requires the exact sequence of machine and human actions, not just a final decision.
An exportable evidence pack makes the runtime trace legible to engineering, governance, and process owners without changing the underlying storage model.
Current Scenario

The demo stays on a single enterprise workflow: invoice approval with agent participation. The seeded runs include a normal path, a policy escalation that ends with human approval, and a blocked attempt followed by human rejection and manual hold.

The signature surface is replay: agent step, policy gate, authority handoff, business consequence, final outcome.
