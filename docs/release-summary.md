## GitHub Release Notes

Agent Activity Graph is a reference implementation for a missing layer in enterprise AI systems: runtime evidence for agentic business workflows.

It shows why orchestration alone is incomplete once agents act inside real processes. The product centers on a single invoice approval workflow and demonstrates event ingestion, policy evaluation, replay, incident review, and an exportable evidence pack. The strongest point is the replay surface, which makes agent action, policy gates, human intervention, and final business outcome legible in one trace.

## LinkedIn Adaptation

Built a flagship reference product called Agent Activity Graph.

The thesis is simple: enterprise agent systems are still missing a runtime evidence layer. Orchestration can move work, but it usually cannot answer what the agent did, under whose authority, what policy allowed or blocked, where a human stepped back in, and how to replay the run later.

This project stays narrow on purpose: one invoice approval workflow, one policy engine, one replay surface, one exportable evidence pack. The goal was not breadth. The goal was to make the missing layer impossible to ignore.

## Portfolio Blurb
